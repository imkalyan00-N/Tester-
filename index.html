BASIC SQL QUERYING & SYNTAX

3.1 BASIC SQL QUERYING [SELECT AND PROJECT using WHERE Clause] :

UNIT-3 Concepts of SQL

☑ The SELECT statement in SQL is used to query data from a database.

☑ The syntax can be quite flexible depending on the complexity of your query.

✓ Here is the complete syntax of the SELECT statement in DBMS:

Complete syntax of SELECT:
SELECT [DISTINCT] column1, column2,
FROM table_names
[WHERE condition]
[GROUP BY column1, column2, ...]
[HAVING condition]
[ORDER BY column1, column2, ... [ASC|DESC]]

Clear Telugu Explanation

SQL lo data ni retrieve cheyyadaniki SELECT statement vaduthamu. Paina unna syntax chuste, manam ye columns kavalo (SELECT), ye table nunchi kavalo (FROM), mariyu elanti conditions (WHERE, GROUP BY etc.) pettalo ee format chepthundi.

2

EXPLANATION OF EACH PART (1-4)

Explanation of each part:

  1. SELECT:

    Specifies the columns that you want to retrieve. You can select one or more columns, or even use * to select all columns.

  2. DISTINCT (optional):

    Used to eliminate duplicate values in the result set. Only unique rows will be returned.

  3. column1, column2, ...:

    Specifies the list of columns you want to retrieve data from.

  4. FROM table_name:

    Specifies the table from which to retrieve the data.

Clear Telugu Explanation

Ikkada prathi keyword exact meaning undi. SELECT ante manaki kavalsina columns ivvadam (* pedithe anni columns vastayi). DISTINCT vadithe duplicate data raadu. FROM ante data ye table nunchi theeskuntunnamo cheppadam.

3

EXPLANATION OF EACH PART (5-8)

  1. WHERE condition (optional):

    Filters the records based on a specified condition. Only rows that satisfy the condition will be included in the result.

  2. GROUP BY column1, column2,... (optional):

    Groups rows that have the same values in specified columns. It is typically used with aggregate functions like COUNT(), SUM(), AVG(), etc.

  3. HAVING condition (optional):

    Filters the grouped records (after aggregation). It is used with GROUP BY to filter groups based on a condition.

  4. ORDER BY column1, column2, ... [ASC|DESC] (optional):

    Sorts the result set based on one or more columns. You can specify ascending (ASC, which is default) or descending (DESC) order.

ARITHMETIC & LOGICAL OPERATIONS

3.2 Arithmetic and logical operations:

In SQL, you can perform arithmetic and logical operations to manipulate data and filter records.

Here's a breakdown of both types of operations:

1. Arithmetic Operations in SQL

Arithmetic operations allow you to perform calculations on numeric data. The basic arithmetic operators in SQL are:

  • + (Addition)
  • - (Subtraction)
  • * (Multiplication)
  • / (Division)
  • % (Modulo-remainder of division)

LOGICAL OPERATIONS

2. Logical Operations in SQL

Logical operations in SQL are used to filter records based on boolean logic. The basic logical operators in SQL are:

  • AND: Combines two conditions; both must be true for the record to be included.
  • OR: Combines two conditions; at least one must be true for the record to be included.
  • NOT: Reverses the truth value of a condition.
  • BETWEEN: Tests if a value is within a range of values.
  • IN: Tests if a value matches any value in a list of values.
  • LIKE: Used for pattern matching in strings.

Clear Telugu Explanation

Logical operators ni WHERE clause tho vaadi records filter cheyadaniki vaduthamu. AND (rendu conditions true avvali), OR (okati true ayina chalu), BETWEEN (range kosam), IN (list kosam), mariyu LIKE (letters pattern checking kosam) vadataru.

LOGICAL EXAMPLES: AND, OR, NOT

Example of Logical Operations:

AND Operator: This query returns employees whose salary is greater than 50,000 and who work in the "Sales" department.

SELECT first_name, last_name, salary, department
FROM employees
WHERE salary > 50000 AND department = 'Sales';

OR Operator: This query returns employees who are in either the "Sales" or "Marketing" departments.

SELECT first_name, last_name, salary, department
FROM employees
WHERE department = 'Sales' OR department = 'Marketing';

NOT Operator: This query returns employees who do not work in the "HR" department.

SELECT first_name, last_name, department
FROM employees WHERE NOT department = 'HR';

LOGICAL EXAMPLES: BETWEEN, IN, LIKE

BETWEEN Operator: This query returns employees who are in either the "Sales", "Marketing", or "Finance" departments.
(Note: The query below demonstrates BETWEEN for salary, though description mentions departments)

SELECT first_name, last_name, salary
FROM employees
WHERE salary BETWEEN 40000 AND 60000;

IN Operator:

SELECT first_name, last_name, department
FROM employees
WHERE department IN ('Sales', 'Marketing', 'Finance');

LIKE Operator (for pattern matching): This query returns employees whose first name starts with the letter "J". The % symbol is a wildcard that matches zero or more characters.

SELECT first_name, last_name
FROM employees
WHERE first_name LIKE 'J%';

COMBINING OPERATIONS & SQL FUNCTIONS

Combining Arithmetic and Logical Operations

You can combine both arithmetic and logical operations in a single SQL query to perform more complex queries.

Example:
This query calculates a 10% bonus (salary * 0.1) for employees in the "Sales" department who have a salary greater than 50,000.

SELECT first_name, last_name, salary, (salary * 0.1) AS bonus
FROM employees
WHERE salary > 50000 AND department = 'Sales';

Example 2:

SELECT product_name, price, (price * 0.15) AS tax
FROM products
WHERE price BETWEEN 50 AND 200 AND price * 0.15 > 10;
