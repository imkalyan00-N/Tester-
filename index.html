6. Assertions (Note: Not supported in Oracle)

Oracle does not support CREATE ASSERTION, unlike the SQL standard.

☐ Workaround:
Use triggers to simulate assertions.

Example: To enforce a constraint across multiple tables, write a BEFORE INSERT or BEFORE UPDATE trigger.

Example using Trigger (simulating assertion):

CREATE OR REPLACE TRIGGER check_total_salary
BEFORE INSERT OR UPDATE ON employees
FOR EACH ROW
DECLARE
total_salary NUMBER;
BEGIN
SELECT SUM(salary) INTO total_salary FROM employees;
IF total_salary + :NEW.salary > 1000000 THEN
RAISE_APPLICATION_ERROR(-20001, 'Total salary cannot exceed 1 million');
END IF;
END;

3.6 NESTED QUERIES

3.6 NESTED QUERIES :

☑ A nested query (also known as a subquery) is a query embedded inside another query.

☑ It is useful when you need to retrieve data that depends on the result of another query.

☑ The inner query executes first and its result is used by outer query.

☑ Nested queries are often used in the WHERE, HAVING, FROM, or SELECT clauses of a SQL statement.

Key characteristics of nested queries:

  • Inner query runs before outer query.
  • Results of inner query for comparison of input data.
  • Nested queries are particularly useful for breaking down complex operations into smaller manageable parts, making it easy to retrieve specific results from database.

Advantages of Nested Queries :

  • Simplifies complex queries, enhances flexibility and improves readability.
  • Commonly used for filtering data, performing calculations or joining database datasets indirectly.

Types of Nested Queries:

  1. Non-Correlated Subqueries: The inner query can be executed independently of the outer query.
  2. Correlated Subqueries: The inner query depends on values from the outer query. It is evaluated once for each row in the outer query.

Correlated Nested Query: The inner query depends on the outer query for its execution for each row processed by the outer query. The operators used in correlated nested queries are exists, not exists.

STRUCTURE & SET OPERATORS

Basic Structure of a Nested Query:

SELECT column1, column2
FROM table1
WHERE column1 = (SELECT column1 FROM table2 WHERE condition);

Set comparison operators allow you to compare a value with a set of values returned by a subquery. These operators are particularly useful when you need to check for membership in a set or compare values against a set.

Set Comparison Operators:

  • = ANY / = SOME: Matches if the value is equal to any of the values returned by the subquery.
  • IN: Matches if the value is contained in the set returned by the subquery.
  • = ALL: Matches if the value is equal to all the values returned by the subquery.
  • NOT IN: Matches if the value is not contained in the set returned by the subquery.
  • > ANY, < ANY, >= ANY, <=ANY: Matches if the value is greater than, less than, etc., any of the values returned by the subquery.

3.7 Sub Queries

A subquery or inner query or nested query is a query within another SQL query and embedded within the WHERE clause.

Subquery is used to return the data that will be used in the main query as a condition to further restrict the data to be retrieved.

Subqueries can be used with the SELECT, INSERT, UPDATE and DELETE statements along with the operators, =, >=, <=, <, >, IN, BETWEEN, etc

Subqueries are a powerful feature in SQL that can simplify complex queries by breaking them into smaller, more manageable pieces.

What is nested sub-query?

3.8 GROUPING & HAVING

3.8 Grouping:

✓ The SQL GROUP BY clause is used in collaboration with the SELECT statement to arrange identical data into groups.

☑ This GROUP BY clause follows the WHERE clause in a SELECT statement and precedes the having clause.

☑ The GROUP BY clause is used to group rows that have the same values in specified columns, often used with aggregate functions like SUM(), COUNT(), AVG(), MIN(), and MAX().

Syntax:
SELECT column1, AGG_FUNC(column2)
FROM table_name
GROUP BY column1;

☑ Taking Sailors, Boats, Reserves tables

Boats (BID, bname, Color)

2. Average salary > 50000

SELECT department_id, AVG(salary)
FROM employees
WHERE job_title = 'Engineer'
GROUP BY department_id
HAVING AVG(salary) > 50000;

First filters engineers, groups them by department, and then filters only those departments where the average salary is more than 50,000

Having:

The Having clause is used along with the Group by clause. The Having clause can be used to select and reject row groups.

The format of the Having clause is similar to the Where clause consisting of the keywords Having followed by a search condition.

The Having clause thus specifies a search condition for groups

Syntax:
select (list of columns, aggregate function) from <tablename>
group by column name
having (condition);

3.9 AGGREGATE FUNCTIONS

3.9 Aggregate Functions:

Aggregate function:

  • It is a function where the values of multiple rows are grouped together, has input on certain criteria to form single value of more significant value.
  • They are used in select and having causes.
  • Returns a single result on group of rows.

The aggregate functions are MAX(), MIN(), SUM(), AVG(), COUNT(), COUNT(*).

Aggregate functions are:

  • MAX(): It returns the max value in given columns.
  • MIN(): It returns the min value in given columns
  • SUM(): It returns sum of all numeric values in givencolumn
  • AVG(): It returns the average of all values in given.columns.
  • COUNT(): It returns the total number of all values in the given column (excluding null values).
  • COUNT(*): It returns the total number of all rows in the given table (including null values).
