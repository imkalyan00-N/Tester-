🔌 Accessing I/O Devices

English Text + Tenglish Mass Explanation (Part by Part)

"Accessing I/O Devices
The I/O devices are connected by using a single bus which enables data transaction between each device as shown in Fig 5.1.
• Bus consists of 3 sets of lines used to carry address, data and control signals.
• Each I/O device is assigned a unique set of address. When the processor places a particular address on the address lines, the device that recognizes this address responds to the commands issued on the control lines.
• The processor requests either a read or a write operation which is transferred over the data lines. When I/O devices and the memory share the same address space, the arrangement is called memory-mapped I/O. There are two ways to deal with I/O devices."
🚌 Asalu Devices CPU tho ela matladathayi? (Bus Concept)

Mana computer lo keyboard, mouse lanti I/O devices anni CPU ki oka "Bus" dwara connect ayyi untayi. (Bus ante mana RTC bus kaadu, idoka group of wires). Deentlo 3 lines untayi:
1. Address Lines: Inti address laaga. Prathi device ki oka pin code (address) untundi.
2. Data Lines: Data veladaniki, ravadaniki.
3. Control Lines: Command ivvadaniki (Read/Write ani).

CPU evaritho matladalo valla address ni address lines meeda pedutundi. Aa address evaridaithe vaalle pallukuntaru. Appudu data lines dwara CPU "Chaduvu" (Read) leda "Raaayi" (Write) ani order istundi. Okavela ee I/O devices, manam vaade main Memory (RAM) iddari address lu oke area lo unte danni Memory-Mapped I/O antaru.

"5.1.1 Memory mapped I/O
• The Memory and I/O devices are sharing information by a common address-space. Any data transfer instruction (like Move, Load) can be used to exchange information. For example,
Move DATAIN, R0
• This instruction reads data from DATAIN (address of input-buffer associated with Keyboard) & stores them into processor-register R0.
Move R0, DATAOUT
• This instruction sends the contents of register R0 to location DATAOUT, which may be the output data buffer of a display unit or a printer."
🧠 5.1.1 Memory Mapped I/O (Oke illu, Oke rules)

Indulo Memory ki, I/O devices ki Common Address Space (oke pin code area) untundi. Deeni valla pedda advantage enti ante, memory nunchi data thevadaniki manam vaade normal instructions (Move, Load lanti vi) vaadukuni I/O tho kuda muchatlochu. I/O ki antu special commands akkarledu.

💡 Instruction 1: `Move DATAIN, R0`
DATAIN ante Keyboard (Input). Keyboard nunchi data thechi CPU lopala unna R0 ane register lo pettu ani cheppadam.

💡 Instruction 2: `Move R0, DATAOUT`
R0 lo unna data ni theeskelli DATAOUT (ante Display leda Printer) ki pampu ani cheppadam. Normal ga ram lo elaa rasthamo, ikkada anthe!

"5.1.2 I/O mapped I/O
• In I/O mapped I/O, the memory and I/0 address-spaces are different. Special instructions are used for data transfer such as IN and OUT.
• The I/O devices use special I/O address space or memory address space.
• The I/O devices examine the low order bits of the address bus to determine whether they should respond.
• Advantage of separate I/O space: I/O devices deal with fewer address-lines."
🚪 5.1.2 I/O Mapped I/O (Separate illu, Special rules)

Indulo Memory address lu veru, I/O address lu veru. Iddari address areas completely different. Anduke normal ga vaade Move/Load kakunda, IN, OUT lanti Special Instructions vaadali.

Ee I/O devices aa address bus loni chivari bits (low order bits) ni check cheskuni, "Idhi naa address eh na? nenu respond avvala?" ani decide avthayi.
✨ Advantage: I/O space separate ga undadam valla address chinnadi ga untundi, so ee I/O devices takkuva address lines thone deal cheyochu (tensions takkuva).

"• The hardware required to connect an I/O device to the bus as shown in the Fig 5.2
• The address decoder enables the device to recognize its address when this address appears on the address lines.
• The data register holds the data being transferred to or from the processor.
• The status register contains information relevant to the operation of the I/O device.
• Both the data and status registers are connected to the data bus and assigned unique addresses. The address decoder, the data and status registers, and the control circuitry required to coordinate I/O transfers constitute the device's interface circuit."
🛠️ Device ni Bus ki ela kaluputaru? (Interface Circuit)

Oka device ni system bus ki direct ga tagilinchaleru. Daaniki konni hardware parts kavali. Avventante:

  • Address Decoder: Idoka watchman lanti di. Address lines meeda vachina address tana owner da kado gurtupattadaniki idi help chestundi.
  • Data Register: Idi oka dabba lanti di. Velle data ni leda vache data ni hold cheskuni unchutundi.
  • Status Register: Idi I/O device paristhiti ento (ready ga unda? bise ga unda?) cheppe register.

Ee Address decoder + Registers + Control circuitry annitini kalipi okati ga set cheste daanne "Device's Interface Circuit" antaru.

"Program Controlled I/O
• Processor repeatedly checks a status flag to achieve required synchronization between processor & input/output device. (We say that the processor polls the device).
Drawback: The processor wastes its time in checking the status of the device before actual data transfer takes place.

5.1.4 Mechanisms Used For Interfacing I/O Operations:
Interrupt I/O
Synchronization is achieved by having I/O device send a special signal over bus whenever it is ready for a data transfer operation."
⏱️ Program Controlled I/O vs ⚡ Interrupt I/O

1. Program Controlled I/O (Polling):
Indulo CPU urike vachi device ready ga unda leda ani aa "status flag" ni repeated ga check chestune untundi. Deenine "Polling" antaru.
Drawback (Bokka): Asalu data transfer jarige kante mundu, ee status checks kosame CPU tana time antha waste cheskuntundi.

2. Interrupt I/O:
Aa polling problems ni theeseyadanike idi vachindi. Indulo CPU urike vachi check cheyadu. Data theeskovadaniki I/O device ready avvagane.. ade direct ga bus lopalnunchi CPU ki oka "Special Signal" (missed call laaga) pamputundi. Deenivalla synchronization perfect ga, CPU time waste avvakunda jarugutundi!
