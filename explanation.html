MOTOROLA 68000

Interrupt Priority & Vectors

Part 01: Priority Levels (0 to 7)
Interrupt structure
  • The 68000 has eight interrupt priority levels. The priority at which the processor is running at any given time is encoded in three bits of the processor status word (I2, I1, I0) With level 0 being the lowest priority.
  • Levels 1 through 6 are the mask-enabled levels.
  • For example, if you set the mask to 100 then only levels 5, 6 and 7 will be enabled; interrupt ‘levels 1 through 4 are disabled.

🏮 8 Priority Levels & Masking (Evariki Preference?)

68000 processor lo mottham 8 Interrupt Priority Levels (0 nunchi 7 daka) untayi. (Level 0 ki lowest priority). CPU prastutam ea level priority lo undo telupadaniki Processor Status (PS) word lo unna 3 bits (I2, I1, I0) vaadutaru.

Indulo Levels 1 nunchi 6 daka unnavi "Mask-enabled" levels (Ante manam kavalante off cheskovachu).

Masking Example: Okavela nuvvu mask bit ni '100' (ante decimal lo 4) ki set chesav anuko. Appudu aa 4 kante ekkuva unna priority levels (ante 5, 6, mariyu 7) matrame ON ayyi untayi. Aa 4 kante thakkuva unna 1, 2, 3, 4 disable (ignore) aypotayi.
Part 02: Level 7 The Boss
  • An interrupt request is accepted only if its priority is higher than that of the processor, with one exception: An interrupt request at level 7 is always accepted. This is an edge triggered non maskable interrupt.

🏮 Level-7: The Ultimate Boss (Non-maskable)

Sadarana ga oka kotha interrupt vachindante.. Processor prastutam run chestunna priority kante aa kotha interrupt priority 'Higher' (ekkuva) unte ne CPU danni accept chestundi.

Kani deeniki oka Exception undi: Level 7 Interrupt! Idhi vasthe CPU em pani lo unna pakkana petti thappakunda accept chesi theerali. Endukante idi oka "Edge Triggered Non-Maskable Interrupt" (NMI - deenni manam code rasi off leda mask cheyaleru).

Part 03: Auto-Raise Priority
  • When the processor accepts an interrupt request, the priority level indicated in the Processor Status (PS) register is automatically raised to that of the request before interrupt request routine is executed. This requests of equal or lower priority are disabled, except for level-7 interrupts, which are always enabled.

🏮 Level up avvadam (Auto Priority Raise)

Oka kotha interrupt CPU accept cheyagane (ISR run chese kante mundu), CPU tana status register (PS) loni priority level ni.. aa vachina kotha interrupt level entha undo antha (Level up) chestundi.

Deeni valla emavtundi ante, current run ayye interrupt kante thakkuva leda equal priority unnavi anni aypotayi (disable avtayi). Evvadu madyalo disturb cheyadu. (Of course, Level-7 vasthe eppudu open eh).

Part 04: Saving to Stack (A7)
  • The processor automatically saves the contents of the program counter and the processor status word at the time of interruption. The PC is pushed on to the processor stack followed by the PS, using register A7 as the stack pointer.
  • A Return from interrupt instruction, called Return-from-exception(RTE), pops the top element of the stack into the PS and pops the next element into the PC.

🏮 State Save Cheyadam & RTE dwara Tirigi Ravadam

Interrupt vachina ventane CPU tana Program Counter (PC) mariyu Processor Status (PS) ni automatic ga daachi pedutundi. Vetini Processor Stack lopaliki tosthundi (Push chestundi). Ee stack ekkada undo point cheyadaniki Register A7 (Stack Pointer) ni vaadutundi.

Interrupt pani antha aypoyaka venakki ravadaniki RTE (Return-from-exception) ane command vaadutaru. Idi stack nunchi PS ni, aa tarvata PC ni pop (bayataki theesi) chesi.. CPU patadi ekkada aagipoyindo akkadi nunche malli start ayyela chestundi.

Part 05: Vectored Interrupts
  • The 68000 processor uses vectored interrupts. When it accepts an interrupt request, it obtains the starting address of the interrupt-service routine from an interrupt vector stored in main memory.
  • There are 256 interrupt vectors, numbered o through 255. Each vector consists of 32 bits that constitute the required starting address.

🏮 Vectored Interrupts ante enti? (Address list)

Motorola 68000 anedi Vectored Interrupts ni vaadutundi. Ante, CPU oka interrupt accept cheyagane, ISR (Service code) ni start cheyadaniki address kavaliga? Aa starting address ni main memory lo unna "Interrupt Vector" (oka address list lanti di) nunchi theeskuntundi.

Main memory lo mottham 256 interrupt vectors (0 nunchi 255 daka) untayi. Prathi vector lo oka 32-bit address (starting address) store ayyi untundi. Ye interrupt vasthe aa vector lo unna address ki CPU jump avtundi.
