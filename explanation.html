BUS ARBITRATION

Who gets the Bus? (Centralized vs Distributed)

STEP 01: Basics of Arbitration
Bus Arbitration
  • The device that is allowed to initiate data transfers on the bus at any given time is called the bus master.
  • Bus Arbitration is the process by which the next device to become the bus master is selected and the bus mastership is transferred to it.
There are 2 approaches to bus arbitration. They are,
1. Centralized arbitration (A single bus arbiter performs arbitration)
2. Distributed arbitration (all devices participate in the selection of next bus master).

🏮 Bus Master ante Evaru? Arbitration ante Enti?

Oka time lo ye device ayithe bus (wires) ni vaadukuni data pampagala power ni theeskuntundo, daanni "Bus Master" antaru. (Appatiki vade thopu anamata).

Mari ee Bus evariki ivvali? Evari tarvata evaru master avvali? ani decide chese process ne "Bus Arbitration" antaru. Deentlo 2 types unnayi:
1. Centralized: Okkade boss (arbiter) untadu, vade andarni control chestadu.
2. Distributed: Boss evaru leru, devices anni matladukuni evariki kavalante vallu theeskuntaru.

PART A: Centralized Arbitration

STEP 02: Centralized & Daisy Chain
Centralized Arbitration:
  • Here the processor is the bus master and it may grants bus mastership to one of its DMA controller. A DMA controller indicates that it needs to become the bus master by activating the Bus Request line (𝐵 𝑅̅̅ ) which is an open drain line. The signal on Bus Request is the logical OR of the bus request from all devices connected to it.
  • When Bus Request is activated, the processor activates the Bus Grant Signal (BG1) and indicates the DMA controller that they may use the bus when it becomes free. This signal is connected to all DMA controllers using a daisy chain arrangement.
  • If DMA controller 1 requests the bus, it blocks the propagation of Grant Signal to other devices and it indicates to all devices that it is using the bus by activating open collector line, Bus Busy (BBSY). Otherwise it passes the grant downstream by asserting BG2.

🏮 Processor Boss & Daisy Chain Concept

Indulo CPU (Processor) ye eppudu Bus Master (Boss). Vere evarikaina (e.g., DMA controller ki) bus kavalante, vallu CPU ki Bus Request (BR) line dwara request pamputaru.

CPU aa request chusi, "Sare theesko" ani Bus Grant (BG1) signal pamputundi. Ikkada twist enti ante, controllers anni oka line lo (Daisy chain arrangement lo) nilchuni untayi.

Daisy Chain Magic: CPU BG1 (Grant) pampagane, adi first unna Controller 1 daggarki veltundi. Okavela vadike bus kavalante, vadu aa BG1 signal ni pakkodiki vellakunda block chesi, Bus ni theeskuni.. andarki Bus Busy (BBSY) ani board pedatadu. Okavela vadiki bus akkarledanuko, aa BG1 ni pass chesthu pakkavadiki (BG2) pamputadu. Simple ga cheppalante, bhojanaallo line ga vaddinche paddathi!

🏮 Data Transfer: Cycle Stealing vs Block Mode

(Paina unna English notes repeated ga undi, kani concept idi):
Oka DMA controller ki bus dorikaka (Master ayyaka) adi data transfer chestundi. Idi 2 modes lo cheyochu:
1. Cycle Stealing: Koncham koncham data (okka word) pampi malli aagadam.
2. Block Mode: Okkasare mottham data pampeseyadam.

Ee DMA tana data pampadam aypoyaka bus ni vadilestundi (releases the bus). Adi vadileyagane malli Processor (CPU) ne thirigi Bus Master aypotundi.

PART B: Distributed Arbitration

STEP 04: The Democracy System
Distributed Arbitration:
  • Distributed arbitration means that all devices waiting to use the bus have equal responsibility in carrying out the arbitration process, without using a central arbiter.
  • Each device on the bus is assigned a 4 bit Identification number. When one or more devices request the bus, they assert the 𝑆̅ 𝑡̅ 𝑎 𝑟̅̅ 𝑡̅ − 𝐴 𝑟̅̅ 𝑏 𝚤̅ 𝑡̅ 𝑟̅̅ 𝑎 𝑡̅ 𝚤̅ 𝑜̅ 𝑛̅̅ signal and place their 4 bit ID number on four open collector lines, 𝐴 𝑅̅̅ 𝐵 0̅̅̅̅ to 𝐴 𝑅̅̅ 𝐵 3̅̅̅̅
  • A winner is selected as a result of the interaction among the signals transmitted over these lines. The net outcome is that the code on the four lines represents the request that has the highest ID number. The drivers are of open collector type.
  • Hence, if the input to one driver is equal to 1, the input to another driver connected to the same bus line is equal to “0” the bus will be in low-voltage state.

🏮 Distributed Arbitration (Andaru Samaname)

Ikkada Centralized laaga oka pedda Boss evaru undaru. Bus kavalsina devices anni matladukuni okari bus theeskuntayi (Equal responsibility).

Idi ela jarugutundi ante, prathi device ki oka 4-bit ID number untundi. Bus kavalsina devices anni Start-Arbitration ane signal icchi, valla valla 4-bit ID ni 4 lines (ARB0 to ARB3) meeda pedatayi.
Final ga evari ID ayithe andarilo kalla peddado (Highest ID Number), vallade Bus! (Oka rakamga Auction laaga). Ivi open collector lines kavatam valla okadu '0' pettina aa line antha low aypotundi.

STEP 05: The ID Comparison Logic
Example:
  • Assume two devices A and B have their ID 5 (0101), 6(0110) and their code is 0111. Each device compares the pattern on the arbitration line to its own ID, starting from most significant bit(MSB).
  • If it detects a difference at any bit position, it disables the drivers at that bit position and for all lower-order bits. It does this by placing “0” at the input of these drivers. In our example, device “A” detects a difference in line 𝐴 𝑅̅̅ 𝐵 1 .
  • Hence it disables the drivers on lines 𝐴 𝑅̅̅ 𝐵 1 and 𝐴 𝑅̅̅ 𝐵 0̅̅̅̅ . This causes the pattern on the arbitration line to change to 0110 which means that “B” has won the contention.
  • It has the advantage of offering higher reliability, because operation of the bus is not dependent on any single device.

🏮 ID Example (Evadu Gelustadu?)

Super example idi. Device A ID vachi 5 (0101), Device B ID vachi 6 (0110) anuko. Rendu kalisi ARB lines meeda signals isthe aa open collector mechanism valla line meeda 0111 untundi.

Ippudu prathi device aa line lo unna value ni valla ID tho (Left to Right) compare cheskuntayi. Ekkadaina theda vasthe (valla own bit 0 undi, kani line meeda 1 unte) aa device vodipoyinattu realize ayyi... danni next bits ni tana side nunchi 0 (disable) chesestundi.

Mana example lo Device A ki 𝐴 𝑅̅̅ 𝐵 1 daggara Theda vastundi, so adi tana drivers aapesi Device B (0110) ni gelipistundi.

Pedda Advantage Enti? Ikkada CPU lanti oka pedda single boss meeda aadharapadi ledu kabatti.. idi chala Highly Reliable (Ekkada aagipodu).
