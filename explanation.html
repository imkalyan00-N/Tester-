PL/SQL Stored Functions In-Depth

AIM: Program development using creation of stored functions, invoke functions in SQL statements and write complex functions.

Part 1: Theory - Asalu Stored Function ante enti? 🤔

1. Definition & Uses
Stored function ante oka named PL/SQL subprogram. Procedure ki, Function ki main difference entante... Function kachithanga oka single value ni RETURN cheyali (using RETURN statement). Veetini manam direct ga SQL queries (SELECT) lopala vadeskovachu.

Enduku vadatharu? (Advantages & Applications):
  • Code Reusability: Okkasari function rasi save cheste, db lo ekkadinunchaina call cheskovachu.
  • Calculations: Salary computation, Grades ivvadam, calculations cheyadaniki best.
  • Encapsulation: Complex business logic antha okachota dachi pettachu.

2. Rules for using Functions in SQL ⚠️
SQL statement (like SELECT) lopala function vadali ante konni strict rules unnayi:

  • Kachithanga oka single value ye return cheyali.
  • Function lopala DML commands (INSERT, UPDATE, DELETE) undakudadu. Only read operations matrame (SELECT) allow chestundi.
  • Parameters ga only IN (input) matrame ivvali. OUT or IN OUT parameters SQL calls lo pani cheyavu.

Part 2: Pre-requisites (Table Setup) 🛠️

Function raase mundu oka chinna `dept` table create chesi oka record insert chedam. (Note: Function lo dname ani fetch chestunnam kabatti, table creation lo column name ni `dname` ga modify chesanu.)

CREATE TABLE dept (
    deptno INT,
    dname VARCHAR2(20)
); 

INSERT INTO dept VALUES (1219, 'sai');
COMMIT;

Part 3: Creating the Function & Tanglish Breakdown 💻

Step 1: Function Header & Variable Declaration

CREATE OR REPLACE FUNCTION getname(dno NUMBER) 
RETURN VARCHAR2 AS
    fname1 VARCHAR2(30);

Emaindi ikkada?
👉 getname ane oka kotha function srustistunnam. Idi input ga dno (department number) ni theeskuntundi.
👉 Ee function output ga oka string ni isthundi kabatti, RETURN VARCHAR2 ani specify chesam.
👉 Output ni temporay ga store cheskodaniki fname1 ane variable theeskunnam.

Step 2: Execution Body

BEGIN
    SELECT dname INTO fname1 FROM dept
    WHERE deptno = dno;
    
    RETURN fname1;

Emaindi ikkada?
👉 Manam pass chesina input dno vadi, table lo aa id ki matching ga unna dname ni SELECT chestunnam.
👉 Vacchina peru ni INTO fname1 antu mana variable loki veskunnam.
👉 Last ga, aa name ni function nunchi bayataki pampadaniki RETURN fname1; vadam. Function ikkade end aipotundi (data dorikithe).

Step 3: Exception Handling (Error vasthe ela?)

EXCEPTION 
    WHEN no_data_found THEN
        raise_application_error(-20100, 'The dno is not found');
END;
/

Emaindi ikkada?
👉 Key Point: Manam ichina dno asalu table lone lekapothe, SELECT query data theiyaledu. Appudu program crash avvakunda no_data_found exception help chestundi.
👉 raise_application_error vadi, mana own error number (-20100) tho manaki kavalsina message ni user ki chupisthunnam. Deeni valla system errors kakunda, clear messages vasthayi.

Part 4: Calling the Function 🚀

Function ready aipoindi, mari danini ela test cheyali? Oka anonymous PL/SQL block rasi ee function ni call cheddam.

SET SERVEROUTPUT ON;

DECLARE
    fname2 VARCHAR2(30); 
    deptno2 NUMBER(5);
BEGIN
    deptno2 := 1219; -- Manam vethukuthunna department number
    
    /* Function ni call chesi, vacchina result ni fname2 ki assign chestunnam */
    fname2 := getname(deptno2); 
    
    /* Result print chestunnam */
    DBMS_OUTPUT.PUT_LINE('The name is: ' || fname2); 
END;
/
Output: The name is: sai

Simple ga cheppalante: getname(1219) ani pilicham. Function lopala velli vethiki "sai" ane peru thechi, bayata unna fname2 ki ichindi. Manam daanni print chesam!
