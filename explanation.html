🎛️ ENABLING & DISABLING INTERRUPTS

Concept + Tenglish Masterclass

Part 01: The Need for Control
Enabling and Disabling Interrupts
  • The facilities provided in a computer must give the programmer complete control over the events that take place during program execution. The arrival of an interrupt request from an external device causes the processor to suspend the execution of one program and start the execution of another.
  • Because interrupts can arrive at any time, they may alter the sequence of events from the envisaged by the programmer. Hence, the interruption of program execution must be carefully controlled.

🏮 Interrupts ni enduku Control Cheyali?

Oka computer lo programmer chethilo full control undali. Bayata nunchi oka "Interrupt" vachinappudu CPU tana current program ni aapi, kothadi (Interrupt Service Routine) start chestundi.

Ee interrupts ki timing undadu, eppudu padithe appudu vastayi. Deeni valla programmer anukunna 'Sequence' debba tintundi. Anduke, ee interruptions ni pichi pichi ga ranivvakunda chala careful ga control cheyali.

Part 02: The Infinite Loop Problem
  • Let us consider in detail the specific case of a single interrupt request from one device. When a device activates the interrupt-request signal, it keeps this signal activated until it learns that the processor has accepted its request.
  • This means that the interrupt-request signal will be active during execution of the interrupt-service routine, perhaps until an instruction is reached that accesses the device.
  • This activated signal, if not deactivated, may lead to successive interruptions, causing the system to enter into an infinite loop.

🏮 Asalu Problem Enti? (The Infinite Loop)

Oka device CPU ki interrupt signal pampagane vadileyadu... CPU "Nenu chusanu le" ani cheppe daka aa signal ni ON (Activated) lone unchutundi.

Deeni ardham enti? CPU aa problem solve cheyadaniki ISR (Code) run chestunnappudu kuda aa interrupt signal ON lone untundi. Okavela aa signal ni deactivate cheyakapote, CPU malli malli kotha interrupt vachindi anukuni pichekki potundi. Okate pani malli malli chesthu 'Infinite Loop' (Eppatiki aagani loop) lo irukkupotundi.

Solution 01: Programmer's Code
To prevent the system from entering into an infinite-loop because of interrupt, there are 3 possibilities.

First Option
  • The first possibility is to have the processor hardware ignore the interrupt-request line until the execution of the first instruction of the interrupt-service routine has been completed.
  • Then, by using an Interrupt-disable instruction as the first instruction in the interrupt-service routine, the programmer can ensure that no further interruptions will occur until an Interrupt-enable instruction is executed.
  • Typically, the Interrupt-enable instruction will be the last instruction in the interrupt-service routine before the Return-from interrupt instruction. The processor must guarantee that execution of the Return-from-interrupt instruction is completed before further interruption can occur.

🏮 Option 1: Interrupt Disable (Programmer's Way)

Aa infinite loop nunchi thapinchukovadaniki 3 options unnayi.

1st Option (Code dwara): CPU aa ISR (Service routine) loni first line run chese daka vere interrupts ni ignore chestundi. Appudu programmer em chestadu ante.. ISR code loni 1st line ga "Interrupt-disable" ani rasi, CPU evadi mata vinakunda chestadu.

Pani antha aypoyaka, ekkadaite "Return" (venakki vellipo) ani command untundo daani kante mundu line lo "Interrupt-enable" ani rasi malli interrupts ni ON chestadu. So madyalo disturbance undadu!

Solution 02: Auto Disable
Second Option
  • The second option is to have the processor automatically disable interrupts before starting the execution of the ISR.
  • After saving the contents of the PC and the processor status register (PS) on the stack, the processor performs the equivalent of executing an Interrupt-disable instruction. It is often the case that one bit in the PS register, called Interrupt-enable, indicates whether interrupts are enabled.

🏮 Option 2: Automatic ga CPU ne aapeyadam

2nd Option (Auto-Disable): Ikkada programmer kurchuni code rayalsina avasaram ledu. CPU ISR (service) start chese mundu tana antha taane automatic ga interrupts ni disable chestundi.

Adi ela chestundi ante, Processor Status (PS) register lo "Interrupt-enable" ani oka special bit (0 or 1) untundi. CPU mundu aa PC ni PS ni safe ga save chesi, ventane aa bit ni OFF chestundi (disable chestundi). Pani aypoyaka malli ON chestundi.

Solution 03: Edge-Triggered
Third option
  • In the third option, the processor has a special interrupt-request line for which the interrupt-handling circuit responds only to the leading edge of the signal. Such a line is said to be edge-triggered. Processor will receive only one request hence there will not be multiple interruption.

🏮 Option 3: Edge-Triggered Signal

3rd Option (Hardware Level): CPU ki velle special line 'Edge-triggered' ayyi untundi.

Edge-triggered ante enti? Signal ala ON lo continous ga undipodu. Just oka chitti spike (edge) vacchi aagipotundi (Current shock kottinattu). Deeni valla CPU ki signal "okkasare" register avtundi, pichi loop lo padipoye chance (multiple interruptions) undadu.

Summary: The 6 Steps
Sequence of events involved in handling an interrupt-request from a single device is as follows:
  1. The device raises an interrupt request.
  2. The processor interrupts the program currently being executed.
  3. Interrupts are disabled by changing the control bits in the PS (except in the case of edge triggered interrupts).
  4. The device is informed that its request has been recognized, and in response, it deactivates the interrupt-request signal.
  5. The action requested by the interrupt is performed by the interrupt-service routine.
  6. Interrupts are enabled and execution of the interrupted program is resumed.

🏮 Final Summary: Interrupt vaste asalu em jarugutundi? (6 Steps)

  1. Step 1: Device "Naku help kavali" ani interrupt raise chestundi.
  2. Step 2: CPU current chestunna program ni aapesthundi.
  3. Step 3: Vere evaru disturb cheyakunda, CPU PS (Processor Status) lo bits marchi Interrupts ni Disable chestundi.
  4. Step 4: CPU aa device ki "Nen chusanu" ani (Acknowledge) cheptundi. Appudu device signal pampadam aaputhundi (deactivates).
  5. Step 5: Aa device ki kavalsina panini ISR (Interrupt-Service Routine) dwara CPU chesi pedutundi.
  6. Step 6: Pani aypoyindi kabatti, malli Interrupts ni Enable chesi.. antha mundu aapeysina patha program ni katha malli modalu pedutundi (resumes).
