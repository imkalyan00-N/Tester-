PL/SQL Cursors In-Depth (Tanglish)

AIM: Develop programs using features parameters in a CURSOR, FOR UPDATE CURSOR, WHERE CURRENT of clause and CURSOR variables.

Part 1: Theory - Asalu Cursor ante enti? 🤔

1. What is a Cursor?
Cursor ante simple ga oka pointer. Manam SQL query (SELECT) execute chesinappudu, database kontha memory allocate chestundi result ni store cheyadaniki. Aa memory area ni point chese daanne cursor antaru. PL/SQL lo query results ni row-by-row process cheyadaniki idi baaga use avtundi.

Implicit Cursor:

Manam INSERT, UPDATE, DELETE chesinappudu leda single-row SELECT rasinappudu, Oracle thanantha taanu automatically ga create chesukune cursor idi. Manam declare cheyalsina avasaram ledu.

Explicit Cursor:

Multiple rows ni return chese SELECT queries ni handle cheyadaniki manam own ga rase cursor idi. Steps: Declare → Open → Fetch → Close.

2. Advanced Cursor Features:

  • Cursor with Parameters: Manam functions ki parameters pass chesinatte, Cursors ki kuda runtime lo values pass cheyachu. Deeni valla okate cursor ni different values isthu reuse cheskovachu. flexibility perugutundi.
  • FOR UPDATE Cursor: Ee cursor tho manam ye rows aite fetch chesthamo, aa rows ni database lock chestundi. Manam update chese lopu vere evaru ah rows ni modify cheyakunda aaputundi.
  • WHERE CURRENT OF: Idi FOR UPDATE ki best friend. Loop thirugutunnappudu, cursor exact ga ye row daggara undo, aa specific row ni matrame update/delete cheyadaniki ee clause vadtharu. Primary keys petti malla WHERE condition rayalsina avasaram ledu.
  • Cursor Variables (REF CURSOR): Normal cursors static (fix ayyi untayi compile time lone). Kani REF CURSOR dynamic ga untundi. Oka sari declare chesi, different different SELECT queries ni daniki assign cheyachu run-time lo. Ee variables ni parameters laaga vere programs ki kuda pass cheyachu.

Part 3: The PL/SQL Program & Tanglish Breakdown 💻

Ippudu meesichina code ni mukkalu mukkalu ga chesi arthamayyela cheptha.

Step 1: Declaration Section

DECLARE
    /* 1. Cursor with parameter and FOR UPDATE */
    CURSOR emp_cur (p_deptno NUMBER) IS
        SELECT emp_id, salary FROM employees 
        WHERE deptno = p_deptno FOR UPDATE;

    v_empid employees.emp_id%TYPE;
    v_salary employees.salary%TYPE;

    /* 2. Cursor Variable (REF CURSOR) */
    TYPE emp_refcur IS REF CURSOR;
    emp_cv emp_refcur;

Emaindi ikkada?
👉 emp_cur ane cursor declare chesam. Daniki p_deptno ane parameter isthunnam. Anduke adi Parameterized Cursor.
👉 Last lo FOR UPDATE pettaam. Ante ee cursor lagina records annitini evaru touch cheyakunda lock avtay.
👉 v_empid, v_salary variables declare chesam data ni hold cheyadaniki.
👉 TYPE emp_refcur IS REF CURSOR tho oka dynamic cursor type create chesam, and emp_cv ani variable create chesam dani kosam.

Step 2: Execution - Fetch and Update

BEGIN
    /* Open parameterized cursor with parameter value 10 */
    OPEN emp_cur(10);
    LOOP
        FETCH emp_cur INTO v_empid, v_salary;
        EXIT WHEN emp_cur%NOTFOUND;

        /* Update current row using WHERE CURRENT OF */
        UPDATE employees SET salary = salary + 1000 
        WHERE CURRENT OF emp_cur;
    END LOOP;
    CLOSE emp_cur;

Emaindi ikkada?
👉 OPEN emp_cur(10); Ikkada parameter value 10 pass chestunnam. So idi department 10 valla data ni matrame lock chestundi.
👉 LOOP petti data ni okkoti ga FETCH chestunnam variables loki.
👉 Data aipoyaka loop nunchi bayataki ravadaniki EXIT WHEN emp_cur%NOTFOUND vadam.
👉 Asalu twist ikkade undi: UPDATE query lo WHERE CURRENT OF emp_cur anagane, cursor present ye row daggara undo ah row yoka salary ni 1000 penchuthundi. Simple and efficient! Tarwata cursor close chesam.

Step 3: Using REF CURSOR to Display Output

    /* Open cursor variable to display updated data */
    OPEN emp_cv FOR
        SELECT emp_id, salary FROM employees 
        WHERE deptno = 10;
        
    DBMS_OUTPUT.PUT_LINE('Employee ID   Salary');
    LOOP
        FETCH emp_cv INTO v_empid, v_salary;
        EXIT WHEN emp_cv%NOTFOUND;
        DBMS_OUTPUT.PUT_LINE(v_empid || '          ' || v_salary);
    END LOOP;
    CLOSE emp_cv;
END;
/

Emaindi ikkada?
👉 Ippudu Update aina kotha data ni chudataniki REF CURSOR (emp_cv) vaduthunnam. Normal cursors declare lo ne open autay, kani idhi OPEN ... FOR ... syntax tho directly executable section lo open chesi query pass chestunnam. (Idi REF cursor speciality).
👉 Malli same Loop vadi, FETCH chesi, DBMS_OUTPUT vadi terminal lo print chestunnam. Final ga close chesesam.
