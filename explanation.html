Part 01: The Alert Signal
Interrupts
  • There are many situations where other tasks can be performed while waiting for an I/O device to become ready. A hardware signal called an Interrupt will alert the processor when an I/O device becomes ready. It can do so by sending a hardware signal called an interrupt to the processor.

🏮 Interrupt ante Asalu Enti?

Oka I/O device (like printer/keyboard) tana pani purthi cheskuni ready ayye lopu, CPU khaaliga kurchokunda vere panulu cheskune situations chala untayi.

Ala CPU vere pani lo unnappudu, I/O device ready avvagane... CPU ki "Nenu ready ayya!" ani oka alert pampistundi. Aa alert pampinche Hardware Signal ne manam "Interrupt" antamu. (Idhi thupukku mani vache missed call lantidi).

Part 02: ISR & Acknowledge
For example, consider, COMPUTE and PRINT routines. The routine executed in response to an interrupt request is called interrupt-service routine.
  • Transfer of control through the use of interrupts happens. The processor must inform the device that its request has been recognized by sending interrupt-acknowledge signal.

🏮 Interrupt Service Routine (ISR) & Acknowledge

Example: CPU edo calculation (COMPUTE) chestundi, ade time lo Printer nunchi PRINT aypoyindi ani interrupt vachindi anuko. Aa vachina interrupt ki respond avthu, CPU ventane oka kottha program (code) ni run chestundi. Aa program ni manam "Interrupt-Service Routine (ISR)" antamu.

Ila jarigina ventane, CPU tana control ni aa ISR ki ichestundi. Alage I/O device tho.. "Babu nee request nenu chusanu/receive cheskunnanu" ani cheppadaniki CPU oka "Interrupt-Acknowledge" signal ni device ki thirigi pampistundi. (Message seen petti reply isthadi anamata).

Part 03: Subroutine & Latency
One must therefore know the difference between Interrupt Vs Subroutine. Interrupt latency is concerned with saving information in registers will increase the delay between the time an interrupt request is received and the start of execution of the interrupt-service routine.

🏮 Delay Enduku Vastundi? (Latency)

Manam ekkada padithe akkada code raase 'Subroutine' ki, ee 'Interrupt' ki difference thelsukovali.

CPU ki interrupt vachina ventane adi velli ISR ni start cheyadu. Mundu CPU current chestunna pani details ni (registers lo unna information ni) safe ga ekkadaina Save cheyali. Ila information save cheyadaniki time padutundi ga? Aa time valla "Interrupt vachina time ki" mariyu "ISR start ayye time ki" madyalo oka chinna delay (gap) vasthundi.

Part 04: Saving State (PC)
  • Processor is executing the instruction located at address i when an interrupt occurs
Routine executed in response to an interrupt request is called the interrupt-service routine.
  • When an interrupt occurs, control must be transferred to the interrupt service routine.
  • But before transferring control, the current contents of the PC (i+1), must be saved in a known location.

🏮 Program Counter (PC) ni Enduku Save Cheyali?

Idi chala important. CPU address 'i' daggara unna code ni run chestunnappudu anukokunda Interrupt vachindi anuko.

Appudu CPU control ni Interrupt-Service Routine (ISR) ki ivvali. Kani ivvadaniki mundu, CPU chese next pani entante.. tana Program Counter (PC) loni value ni (ante i+1 ni) oka safe place lo save cheyali.

PC (Program Counter) ante enti? CPU tarvata ea line of code (address) run cheyalo gurtupettukune book lanti di. So 'i' aypoyindi kabatti, next 'i+1' ki vellali ani CPU gurtupettukuni danni save chestundi.

Part 05: Return & The Stack

🏮 Interrupt Aypoyaka Malli Ela Vastundi?

Paina 'i+1' ni enduku save chesam? Endukante, ISR (interrupt code) motham ayipoyaka CPU malli venakki ochi patadi continue cheyali ga! Aa "Return-from-interrupt" command ranavvagane, CPU malli correct ga 'i+1' address nunche pani start chestundi.

Mari ee Return address (PC lo unna i+1 value) ekkada dachi pedataru? Usually CPU loni "Processor Stack" meeda daachi pedutundi. (Oka dabba lo vesinatlu).

Part 06: Definition of Latency
Interrupt latency: Delay between the time an interrupt request is received and the start of execution of the interrupt service routine.

🏮 Interrupt Latency (Direct Definition)

Interrupt Latency ante simple ga:
"Device nunchi Interrupt Request CPU ki reach ayina time ki.. mariyu CPU aa problem solve cheyadaniki Interrupt Service Routine (ISR) ni run cheyadam start chesina time ki madyalo unde gappe (Delay)" ee Latency.
