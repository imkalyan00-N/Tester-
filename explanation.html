PL/SQL Procedures: Complete Deep Dive

Theory & Program Explanation in Tanglish

1. AIM (Manam em achieve cheyali anukuntunnam?)

AIM: Programs development using creation of procedures, passing parameters IN and OUT of procedures.

Explanation: Ee concept yokka main target enti ante, database lo Procedures ela create cheyali, vatiki variables (parameters) ni ela pass cheyali, mariyu aa procedure nunchi output (OUT parameters dwara) ela thechukovali anedi nerchukovadam.

2. What is a Procedure? (Asalu Procedure ante enti?)

  • A procedure is a named PL/SQL subprogram that performs a specific task.
  • It can accept parameters, execute SQL/PL-SQL statements, and return results through OUT parameters.

Procedures are mainly used to:

  • Modularize programs
  • Improve reusability
  • Reduce code repetition
  • Improve performance and security
Explanation:
Procedure ante oka peru unna PL/SQL code block (subprogram). Idi oka specific pani cheyadaniki vadatham.

Enduku vadali?:
  • Modularize programs: Pedda code ni chinnna chinna mukkalu (modules) ga vidadeeyadam.
  • Improve reusability & Reduce code repetition: Oka sari code rasi procedure create cheste, malli malli kotha code rayalsina avasaram ledu. Aa procedure peru tho call chesukunte chalu. Idi code repetition ni thaggisthundi.
  • Improve performance & security: Anni SQL statements okesari database ki velthayi kabatti speed peruguthundi (network traffic thagguthundi). Alage tables direct ga access ivvakunda, procedure dwara access istham kabatti security kooda untundi.

3. Syntax for Creating a Procedure

CREATE OR REPLACE PROCEDURE procedure_name ( parameter_name mode datatype, parameter_name mode datatype ) AS BEGIN -- executable statements END; /
Explanation:
CREATE OR REPLACE: Kotha procedure create cheyadaniki CREATE. Oka vela same peru tho already unte dantlo changes update cheyadaniki REPLACE vadatham.
Tarvatha brackets () lopaliki manam parameters (variables) pass chestham. Prathi parameter ki 3 vishayalu undali: Name, Mode (IN/OUT), and Datatype (number/varchar2).

4. Parameter Modes (Parameters lo rakalu)

IN Parameters

  • Used to pass values to the procedure
  • Read-only inside the procedure
  • Cannot be modified

Tanglish: Idi just input ivvadaniki. Bayata nunchi value lopaliki theeskelladaniki matrame. Lopala aa value ni manam change cheyalem (read-only). Default ga em ivvakapothe adi IN parameter e.

OUT Parameters

  • Used to return values from the procedure
  • Value must be assigned inside the procedure
  • Cannot accept input values

Tanglish: Procedure lopala calculate ayina result ni bayataki theeskuradaniki idi vadatham. Idhi input ni theeskodu, lopala unna result ni hold cheskuni bayataki isthundi.

IN OUT Parameters

  • Used for both input and output
  • Value can be modified and returned

Tanglish: Idi 2-in-1. Modata input theeskuntundi. Lopala manam dani meeda calculations chesi same variable thone kotha value ni bayataki pampeyochochu.

5. Calling a Procedure (Procedure ni ela pilavali?)

-- Method 1: Using EXEC EXEC procedure_name(parameters); -- Method 2: Inside a PL/SQL block BEGIN procedure_name(parameters); END; /
Explanation: Create chesina procedure run avvali ante danini pilavali (call cheyali). SQL Plus prompt paina unte simple ga EXEC vadi pilavachu. Leda inko program nunchi call cheyali ante BEGIN ... END; (Anonymous block) lopala direct ga peru thote pilavachu.

6. Advantages & Procedure vs Function

Advantages of Procedures:

  • Improves program structure
  • Enhances reusability
  • Reduces network traffic
  • Improves security by hiding business logic
  • Easy maintenance and debugging
Explanation: Code structure neat ga untundi. Oka sari rasi enni sarlaina vadukovachu. Client nunchi prathi chinna query velle badulu okesari procedure call velthundi kabatti network speed baguntundi. Asalu lopala em logic undo bayata user ki teliyadu kabatti security! Errors ochina easily fix cheyachu (maintenance).

Difference Between Procedure and Function

Procedure Function
May or may not return a value Must return a value
Uses OUT parameters to return values Uses RETURN statement to return a value
Called using EXEC or inside a block Can be used directly in SQL expressions (like SELECT)
Explanation:
Idi chala important interview question! Procedure ki pakka result ivvali ane rule ledu, isthe OUT parameter tho isthundi. Kani Function matram pakka RETURN keyword vadi oka value ni return chesi theerali. Alage Function ni manam SELECT function_name FROM dual; ani SQL query lopalane direct ga vadukovachu, kani Procedure ni ala vadalem.

7. The Practical Program (Deep Dive)

Step 1: Table Creation & Data Insertion

CREATE TABLE enquiry(enqno1 number, fname varchar2(30)); INSERT INTO enquiry VALUES(111,'sai'); INSERT INTO enquiry VALUES(112,'sindhu');
Explanation: First manam 'enquiry' ane table create chesthunnam. Indulo 2 columns unnayi. enqno1 (Number datatype) mariyu fname (varchar2 datatype). Dani tarvatha rendo record lu insert chesam. Enquiry id 111 ki peru 'sai', 112 ki peru 'sindhu'.

Step 2: The Procedure (`sriganesh`)

CREATE OR REPLACE PROCEDURE sriganesh(enquiryno1 IN number, fname1 OUT varchar2) IS fname2 varchar2(30); BEGIN SELECT fname INTO fname2 FROM enquiry WHERE enqno1 = enquiryno1; fname1 := fname2; EXCEPTION WHEN NO_DATA_FOUND THEN RAISE_APPLICATION_ERROR(-20100, 'The given number is not present'); END sriganesh; /
Line-by-Line Tanglish Explanation:

CREATE OR REPLACE PROCEDURE sriganesh : 'sriganesh' ane peru tho procedure create chesthunnam.

(enquiryno1 IN number, fname1 OUT varchar2) : Ikkada 2 parameters unnayi. Manam number ni (enquiryno1) input ga istham (IN), daniki match ayye peru ni (fname1) procedure manaki output ga isthundi (OUT).

fname2 varchar2(30); : Idi local variable. Ee procedure lopala matrame pani chestundi. Database nunchi vachina data ni temporary ga store cheskodaniki.

SELECT fname INTO fname2 FROM enquiry WHERE enqno1 = enquiryno1; : Manam pass chesina input number (enquiryno1) ni table lo unna enqno1 tho check chesthundi. Match ayithe, aa fname ni thechi manam paina declare chesina temporary variable fname2 loki vesthundi (INTO).

fname1 := fname2; : Ippudu result unna fname2 ni, manam output ivvalsina parameter fname1 loki assign chesthunnam. Idi direct ga program bayataki vellipothundi.

EXCEPTION ... NO_DATA_FOUND : Oka vela manam ichina enquiry number table lo lekapothe SQL error isthundi. Aa error ni handle cheyadaniki idi. Lekapothe RAISE_APPLICATION_ERROR dwara manam own custom error (-20100) ni chupisthunnam (User-defined errors -20000 nunchi -20999 madhya undali).

Step 3: Calling the Procedure (Anonymous Block)

-- Enable DBMS_OUTPUT to see the result SET SERVEROUTPUT ON; -- Anonymous block to call the procedure DECLARE enqno2 number(5); fname2 varchar2(30); BEGIN enqno2 := 111; -- Sample enqno -- Call the procedure sriganesh(enqno2, fname2); -- Output the result to the console dbms_output.put_line('The name is: ' || fname2); EXCEPTION WHEN OTHERS THEN -- Handle any unexpected exceptions and display them dbms_output.put_line('Error: ' || SQLERRM); END; /
Line-by-Line Tanglish Explanation:

SET SERVEROUTPUT ON; : PL/SQL lo emaina print chesedi screen meeda kanipinchali ante idi pakka ON cheyali. Leka pothe "PL/SQL procedure successfully completed" ani vasthundi kani result kanapadadu.

DECLARE enqno2 number(5); fname2 varchar2(30); : Procedure ki values pass cheyadaniki, akkada nunchi vache value ni theeskodaniki ikkada kotha variables declare chesam.

enqno2 := 111; : Manam search cheyalani anukuntunna enquiry number (Input).

sriganesh(enqno2, fname2); : Idhe main Procedure Call! Manam procedure ki enqno2 (111) ni pampisthunnam. Procedure run ayyaka, akkada assign ayina OUT parameter direct ga vachi ikkada unde fname2 lo kurchuntundi.

dbms_output.put_line(...) : Vachina result ni console lo print chesthunnam. So output "The name is: sai" ani print avthundi.

EXCEPTION WHEN OTHERS THEN : Oka vela procedure call lo emaina error vasthe (example manam 113 ani pass cheste, procedure lo data ledu ani exception vasthundi kada), adi ikkada catch ayyi error message (SQLERRM) ni print chesthundi program crash avvakunda.
