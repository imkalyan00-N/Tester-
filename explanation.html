PART A: ARM Processor Structure

STEP 01: ARM Intro
ARM Interrupt Structure
5.3.1.1 Introduction to ARM
  • A collection of Reduced Instruction Set Computer (RISC) instruction set architectures for computer processors that are tailored for different contexts is known as ARM (stylized in lowercase as an arm; originally an abbreviation for Advanced RISC Machines.
  • For light, portable, battery-powered devices like smartphones, laptops, and tablet computers, as well as other embedded systems, ARM processors are preferred because they are less expensive, consume less power, and generate less heat than their competitors.

🏮 Asalu ARM ante enti? (Smartphones raju)

ARM (Advanced RISC Machines) anedi oka RISC (Reduced Instruction Set Computer) architecture. Ante simple and fast instructions vaade processor idi.

Idi main ga mana Smartphones, Tablets, chinna laptops lo vaadutaru. Enduku? Endukante ARM processors Rate takkuva (less expensive), Battery takkuva tagutayi (consume less power), mariyu Heat asalu avvavu (generate less heat). Intel lanti pedda processors tho compare cheste idhe deeni super power.

STEP 02: ARM Modes
  • Modes of operation
    ARM processor has 7 modes of operation.
     Switching between modes can be done manually through modifying the mode bits in the CPSR register.
     Most application programs execute in user mode
     Non user modes (called privileged modes) are entered to serve interrupts or exceptions
     The system mode is special mode for accessing protected resources. It don‘t use registers used by exception handlers, so it can‘t be corrupted by any exception handler error.

🏮 7 Modes of Operation (7 avataralu)

ARM processor ki mottham 7 Modes untayi. Aa modes madyalo switch avvalante CPSR ane register loni bits ni marchali.

1. User Mode: Normal apps anni deentlone run avtayi.

2. Privileged Modes: Interrupts / Exceptions vachinappudu CPU ee mode loki veltundi.

3. System Mode: Idi chala special. Protected files/resources ni access cheyadaniki vaadtaru. Idi exception handlers vaade registers ni muttukodu, kabatti secure ga untundi.

STEP 03: ARM Registers
  • ARM register set
     ARM processor has 37 32-bit registers.
     31 registers are general purpose registers.
     6 registers are control registers
     Registers are named from R0 to R16 with some registers banked in different modes
     R13 is the stack pointer SP (Banked)
     R14 is subroutine link register LR (Banked)
     R15 is progrm counter PC
     R16 is current program status register CPSR (Banked)

🏮 37 Registers & Vati Perlú

ARM lo mottham 37 (32-bit) registers unnayi. Andulo 31 General purpose, 6 Control registers. Veetiki R0 nunchi R16 ani perlu pettaru.

  • R13 = SP (Stack Pointer - Stack memory ni point chestundi)
  • R14 = LR (Link Register - Subroutines nunchi return address save chestundi)
  • R15 = PC (Program Counter - Next run ayye instruction address)
  • R16 = CPSR (Current Program Status Register - Processor mode ni status ni save chestundi)
STEP 04: Exceptions Source
Sources for Exceptions in ARM
  • The term exception is used to refer to any event that causes an interruption
  • ARM processor has five sources for exceptions, only two of which are external interrupt request lines, IRQ(Interrupt ReQuest) and FIQ (Fast Interrupt reQuest).
  • There is one software interrupt instruction, SWI and two exceptions that may be caused by abnormal conditions encountered during program execution. These exceptions are an external abort following a bus error and an attempt to execute an undefined instruction.

🏮 5 Sources of Exceptions (CPU ki Disturbance ekkadi nunchi vastundi?)

'Exception' ante simple ga CPU pani ni aape (interrupt chese) oka event. ARM lo ila 5 sources nunchi interrupts vastayi:

  1. IRQ: Normal Hardware Interrupt
  2. FIQ: Fast Hardware Interrupt (Chala speed ga respond avvali)
  3. SWI: Software Interrupt (Code dwara vache interrupt)
  4. Data/Bus Abort: Data chaduvutunapudu error vasthe
  5. Undefined Instruction: CPU ki ardham kani code vaste
STEP 05: Reset & Abort
  • The Reset condition is included in this structure because it must override all other conditions to bring the processor to a known starting condition.
  • There are two abort conditions. Data Abort arises from an error in reading or writing data, and Prefetch abort arises from an error when prefetching instructions from the memory

🏮 Reset mariyu Abort ante enti?

Reset: Reset anedi list lo boss lanti di. Idi vachindante migathavanni cancel aypoyi, processor antha fresh ga first nunchi start avtundi.

Abort conditions: Ivi rendu rakalu. Data Abort (Data read/write chesetappudu error vaste idhi vastundi). Prefetch Abort (Memory nunchi next code thecchukune tappudu error vaste idi vastundi).

STEP 06: Initialization & CPSR
All interrupts are disabled on startup for the ARM CPU until the initialization code turns them on.
  • The Processor Status Registers’ bit can be changed to enable or disable the interrupts (PSR or CPSR where C stands for current).
  • The CPSR also determines whether the processor is decoding Thumb instructions and the processor mode (SVC, System, User, etc.).
  • The application can read and write to the CPSR in its entirety when operating in privileged mode, but it can only read the CPSR when operating in non-privileged mode.
  • The processor enters the appropriate interrupt or exception mode in response to an interrupt or exception, which causes a portion of the main registers to be banked, swapped out, or replaced with set mode registers.
  • The interrupt masks’ ability to be enabled and disabled is controlled by bits. The two interrupt inputs on the ARM processor can both be regarded as general-purpose interrupts. Interrupt Request (IRQ) and Fast Interrupt Request are the names of the first and second, respectively (FIQ)

🏮 Startup & CPSR Power (Enabling Interrupts)

System on avvagane (startup lo) anni interrupts disable ayye untayi. Initialization code (boot ayyaka) matrame vaatini ON chestundi.

Interrupts ON/OFF cheyali anna, processor ey mode lo (User/System) undalo decide cheyali anna CPSR (Current Program Status Register) ye main!

Rule: Nuvvu Privileged mode lo unte CPSR ni chadavochu, marchochu (Read/Write). Ade Non-privileged mode lo unte just chadavagalam anthe (Read-only). Interrupt vachinappudu konni registers "Banked" (swapped/replaced) aypotayi. IRQ, FIQ anevi general purpose hardware interrupts.
STEP 07: Priority Structure
The regular sequential execution of instructions can be stopped by one of seven events on the ARM processor. Since not all events are created equal, the processor must adopt a priority strategy because these events may occur simultaneously.
  • Exceptions are handled according to the following priority structure
    1. Reset (Highest Priority)
    2. Data Abort
    3. FIQ
    4. IRQ
    5. Prefetch Abort
    6. Undefined Instruction (Lowest Priority)

🏮 Priority List (Evariki First Preference?)

Oke sari rendu moodu interrupts vaste evaridi mundu solve cheyali? Daanike ee Priority List. (Idi exam lo chala important).

1. Reset (VVIP - Highest Priority)
2. Data Abort
3. FIQ (Fast Interrupt)
4. IRQ (Normal Interrupt)
5. Prefetch Abort
6. Undefined Instruction (Lowest Priority)

PART B: Motorola 68000 Structure

STEP 08: 68000 Intro
268000 Interrupt Structure
5.3.2.1 Introduction
  • Motorola 68000 is a 16/32-bit complex instruction set computer (CISC) Microprocessor.
  • It has 32-bit data and address registers. Externally the processor has 16-bit data bus and 24-bit address bus, which limits the size of addressable memory to 16 MB.
  • It uses a 16-bit data arithmetic logic unit (ALU). The Register architecture of 68000 is shown in Fig 5.9.
  • The data registers can be used to handle (8-bit) bytes, (16-bit) words, or 32-bit long words.
  • There are seven general purpose Address registers (A0-A6). These registers can handle either 16-bit word or 32-bit long word operands.

🏮 Motorola 68000 Basics (CISC Processor)

ARM RISC ayte, ee Motorola 68000 anedi CISC (Complex Instruction Set) microprocessor.

Deeni lopala (internal ga) 32-bit registers unna kuda... Bayataki (externally) matram 16-bit data bus, 24-bit address bus matrame untayi. Deeni valla idi maximum 16 MB memory ni matrame access cheyagaladu. Deeni ALU (Arithmetic Logic Unit) kuda 16-bit eh!

Indulo Data Registers 8-bit, 16-bit leda 32-bit data ni handle chestayi. Address Registers (A0 to A6) kuda 16/32-bit ni handle chestayi.

STEP 09: 68000 Status Register
  • A 16-bit status register is divided into two 8-bit bytes : The system byte and the user byte. Fig. shows the bit assignments for the status register. The system byte of the status register contains status information that is system-related. The user byte, on the other hand, contains the condition code status bits (X, N, Z, V and C) which are instruction or program related. Bits in the system byte of the Status register can only be altered when the MC 68000 is in the supervisor mode.
    Bit 13 of the status register is the S-bit, which specifies whether the MC68000 is in the supervisor or user mode of operation. When the bit is 1, the MC68000 is in the supervisor mode, and when it is 0 the microprocessor is in the user mode.
  • The most significant bit of the status register is the Trace (T) flag. If this bit is 0 then the MC68000 operates normally. However, if the bit is 1, the microprocessor is in the trace mode of operation. After each instruction is executed in the trace mode, a trap is forced so that a debugging program can monitor the results of that instruction’s execution.

🏮 16-bit Status Register (System vs User byte)

Indulo oka 16-bit Status register untundi. Deenni rendu 8-bit mukkalu ga chesaru: 1. System Byte, 2. User Byte.

User byte lo progams ki kavalsina flags (X, N, Z, V, C) untayi. System byte lo system info untundi (Danni marchalante compulasary 'Supervisor mode' lo undali).

S-Bit (13th Bit): Idi 1 unte 'Supervisor Mode' (Full powers), 0 unte 'User Mode' (Normal powers) ani decide chestundi.

T-Flag (Trace Flag - Highest Bit): Idi 0 unte system normal ga run avtundi. Idi 1 unte 'Trace Mode' loki veltundi. Trace mode ante debugging kosam prathi chinna instruction ayyaka system ni trap (pause) chestundi, check cheskovadaniki.
