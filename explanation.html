I/O DEVICES

Full Text + Step-by-Step Tenglish Explanation

STEP 1
Accessing I/O Devices
The I/O devices are connected by using a single bus which enables data transaction between each device as shown in Fig 5.1
  • Bus consists of 3 sets of lines used to carry address, data and control signals.

🏮 Asalu Devices ni Ela Connect Chestaru? (The Bus)

Mana I/O devices (Keyboard, mouse, printer lantivi) anni CPU tho matladadaniki oka "Single Bus" (kone wires group) dwara connect ayyi untayi. Deeni dwara data theeskovadam, ivvadam jarugutundi.
Ee Bus lo main ga 3 lines untayi:
1. Address Lines: Device address theeskelladaniki.
2. Data Lines: Asalu data ni theeskelladaniki.
3. Control Lines: Commands (Read/Write) theeskelladaniki.

STEP 2
  • Each I/O device is assigned a unique set of address. When the processor places a particular address on the address lines, the device that recognizes this address responds to the commands issued on the control lines.
  • The processor requests either a read or a write operation which is transferred over the data lines. When I/O devices and the memory share the same address space, the arrangement is called memory-mapped I/O. There are two ways to deal with I/O devices.

🏮 CPU Device tho Ela Matladutundi?

Prathi I/O device ki oka Unique Address (Pin-code) untundi. CPU aah address lines meeda oka address pettagane, evari address aythe match avtundo aah device palukutundi. Appudu CPU control lines dwara command isthundi (Data theesko leda Data ivvu ani).

Oka vela mana I/O devices, mariyu main Memory (RAM) iddari address lu oke space (Area) lo kalisipoyi unte, daanni "Memory-Mapped I/O" antaru. Ee I/O devices tho deal cheyadaniki 2 ways unnayi. Avi kinda chuddam.

STEP 3
5.1.1 Memory mapped I/O
  • The Memory and I/O devices are sharing information by a common address-space. Any data transfer instruction (like Move, Load) can be used to exchange information. For example,
    Move DATAIN, R0
  • This instruction reads data from DATAIN (address of input-buffer associated with Keyboard) & stores them into processor-register R0.
    Move R0, DATAOUT
  • This instruction sends the contents of register R0 to location DATAOUT, which may be the output data buffer of a display unit or a printer.

🏮 Memory Mapped I/O (Oke Area - Normal Commands)

Indulo Memory (RAM) mariyu I/O devices iddariki okate Common Address Space (oke intlo unnattu) untundi. Deeni valla manam I/O tho matladataniki special commands akkarledu. RAM nunchi data theadaniki vaade normal commands (Move, Load) vaadithe chalu.

Example 1: Move DATAIN, R0
Idi Keyboard (DATAIN) nunchi vachina data ni chadivi CPU loni R0 register lo pettu ani cheptundi.

Example 2: Move R0, DATAOUT
Idi R0 register lo unna data ni theeskelli Display leda Printer (DATAOUT) ki pampu ani cheptundi.

STEP 4
5.1.2 I/O mapped I/O
  • In I/O mapped I/O, the memory and I/0 address-spaces are different. Special instructions are used for data transfer such as IN and OUT.
  • The I/O devices use special I/O address space or memory address space.
  • The I/O devices examine the low order bits of the address bus to determine whether they should respond.
  • Advantage of separate I/O space: I/O devices deal with fewer address-lines.

🏮 I/O Mapped I/O (Separate Area - Special Commands)

Indulo Memory address lu veru, I/O address lu veru (Different Address Space). Separate illu anamata. Anduke Ikkada normal 'Move' panicheyadu, IN mariyu OUT lanti special commands matrame vaadali.

I/O devices address bus loni chivari (low order) bits ni check chesi adi vaalla address auna kada ani gurtupadatai.
Advantage: Ee I/O address space separate ga undatam valla, idi chinna size lo untundi. So, I/O devices takkuva address-lines thone pani kaniccheyochu.

STEP 5
  • The hardware required to connect an I/O device to the bus as shown in the Fig 5.2
  • The address decoder enables the device to recognize its address when this address appears on the address lines.
  • The data register holds the data being transferred to or from the processor.
  • The status register contains information relevant to the operation of the I/O device.

🏮 Device ni Bus ki Kalupe Hardware Parts

Oka I/O device ni bus ki connect cheyadaniki konni hardware parts kavali. Avi entante:

  • 1. Address Decoder: Address lines meeda vachina address tana owner dho kado (Idi naa address ea na?) ani gurtupattadaniki idi help chestundi. (Watchman laga)
  • 2. Data Register: CPU nunchi vache data ni leda CPU ki velle data ni temporary ga daachi unche box idi.
  • 3. Status Register: Device ready ga unda? leda busy ga unda? ani current condition (operation) gurinchi cheppe register idi.
STEP 6
  • Both the data and status registers are connected to the data bus and assigned unique addresses. The address decoder, the data and status registers, and the control circuitry required to coordinate I/O transfers constitute the device's interface circuit.

🏮 Interface Circuit ante enti?

Data register mariyu Status register ee rendu kooda Data bus ke connect ayyi untayi, vatiki kuda unique address untundi.

Paina cheppina Address decoder + Data register + Status register + Control circuitry... ee annitini kalipithe ayye total set ne "Device's Interface Circuit" antaru.

STEP 7
Program Controlled I/O
  • Processor repeatedly checks a status flag to achieve required synchronization between processor & input/output device. (We say that the processor polls the device).
Drawback: The processor wastes its time in checking the status of the device before actual data transfer takes place.

🏮 Program Controlled I/O (Polling) & Daani Problem

Indulo CPU urike vachi I/O device daggara unna 'status flag' ni malli malli check chestune untundi (Nuvvu ready na? Nuvvu ready na? ani). Deenine "Polling" antaru.

Drawback (Loss): Asalu data theeskovadaniki kante mundu.. device ready ga unda leda ani check cheyadanike CPU tana amulyamaina time ni antha waste cheseskuntundi.

STEP 8
5.1.4 Mechanisms Used For Interfacing I/O Operations:
Interrupt I/O
Synchronization is achieved by having I/O device send a special signal over bus whenever it is ready for a data transfer operation.

🏮 Interrupt I/O (The Smart Solution)

Aa time waste problems ni theeseyadanike idi vachindi. Ikkada CPU eppudu padithe appudu vachi adagadu. Data transfer cheyadaniki I/O device eppudu ready avtundo, appudu ade direct ga CPU ki oka "Special Signal" (Interrupt) pampistundi. Appudu CPU matrame vachi pani chuskuntundi. Idhi chala smart way!
